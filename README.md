**Проект "Моя родина - моя гордость: Виртуальное путешествие по наследию Курской области"**

Данный проект создан для продвижения Курской области в интернете и с целью повышения квалификации автора.

**Деплой**:
1. *Статика - timeweb хостинг.*
2. *Динамика - timeweb сервер.*
3. *Медиа - timeweb S3 хранилище.*

Ссыла на макет сайта в Figma: https://www.figma.com/design/T5IGGj7dyoiJXBTZ5mEdxG/%D0%9A%D1%83%D1%80%D1%81%D0%BA%D0%B0%D1%8F-%D0%BE%D0%B1%D0%BB%D0%B0%D1%81%D1%82%D1%8C?node-id=0-1&t=FfxYrqV6sVgo9Oa9-1

**Управление проектом**
# Backend
    cd backend && docker compose up      # запуск
    cd backend && docker compose down    # остановка

# Frontend
    cd frontend && npm run dev   # разработка
    cd frontend && npm run build # сборка


*Технологии разработки*

1. Frontend
    Разработка ориентирована на создание интуитивно понятного интерфейса (UI) и продуманного пользовательского опыта (UX) на базе библиотеки React.js, фреймворка Next.js. Функционал включает: адаптивную вёрстку (desktop/tablet/mobile), бургер-меню, карточки с описанием, интерактивные карты и форму обратной связи. Особое внимание уделено accessibility (доступности): предусмотрены настройки размера шрифта, переключение между светлой, тёмной темами и режимом для слабовидящих, а также семантическая вёрстка, навигация с клавиатуры. Проект кроссбраузерен и оптимизирован для всех типов устройств.

2. Backend
    В качестве серверной платформы выбран FastAPI. В качестве системы управления базами данных (СУБД) используется PostgreSQL, развертывание - Docker.

    API разрабатывается в соответствии с принципами RESTful архитектуры и спецификацией JSON:API. Это обеспечивает стандартизированный формат обмена данными, предсказуемую структуру запросов и ответов, а также удобство интеграции с клиентскими приложениями.

    Регистрация и аутентификация пользователей реализованы на основе JWT-токенов. Токен сохраняется в httpOnly cookie и автоматически отправляются серверу с каждым запросом.

    Обработка комментариев реализована следующим образом: после отправки комментарий сохраняется в БД со статусом "на модерации". Публикация происходит автоматически через 3 дня без участия администратора. Для пользователей доступна опция подписки на уведомления — в этом случае на указанную почту приходит письмо о выходе комментария. Фоновая отправка писем делегирована Celery и брокер Redis для оптимизации производительности.

    Все медиа от пользователя будут храниться в S3 хранилище.

    Язык разработки - Python

    В системе предусмотрено 2 роли:

    **Пользователь**
    Описание endpoint:
    - Зарегистрироваться на сайте (через подтверждение кода на почте, сервис - SMTP‑сервер)
    - Верифицировать почту
    - Выйти из аккаунта
    - Подписаться на рассылку
    - Отписаться от рассылки
    - Оставить комментарий
    - Разместить картинку в комментарии
    - Удалить комментарий (с картинкой)
    - Просматривать комментарии других пользователей (с пагинацией)

    **Администратор**
    Для администратора реализован интерфейс управления пользователями и контентом: доступен список всех зарегистрированных пользователей, а также функционал модерации комментариев (подтверждение/отклонение).

    Описание endpoint:
    - Получение всех пользователей
    - Верификация комментариев


3. Дальнейшая поддержка
    Для отслеживания посещаемости, поведения пользователей и технического состояния сайта будут подключены системы веб-аналитики: Яндекс.Метрика и Google Analytics. Также планируется регистрация проекта в Яндекс.Вебмастер и Google Search Console для управления индексацией сайта в поисковых системах. Для мониторинга нагрузки на API, логирования и оповещения об ошибках будут использованы Sentry, Prometheus, Grafana и профайлер fastapi-radar.
